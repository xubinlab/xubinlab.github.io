TECH STACK · 技术体系

1. 物理与 CAE 建模（Physics & CAE Modeling）

  • 流体力学基础：可压缩/不可压缩流、边界层、激波、涡结构。
  • 守恒方程与控制方程：质量、动量、能量方程的积分/微分形式。
  • 湍流建模：RANS（k–ε/k–ω/SST）、LES/DES 的工程适用性与局限。
  • 多物理：多相流（VOF / Level-Set / Euler–Euler / Euler–Lagrange）、传热传质、燃烧与反应流。
  • 典型工程场景：风扇、压气机、燃烧室、涡轮等部件的需求差异。

2. 数值方法、求解器与 HPC（Numerics, Solvers & HPC）

  • 离散方法：FVM/FEM/DG 等离散框架，时间推进（显式/隐式、多步法/Runge–Kutta）。
  • 求解器设计：压力–速度耦合、完全耦合、非线性迭代与收敛判据。
  • 稀疏线性系统：Jacobian 结构、预条件、CG/BiCGStab/GMRES 等 Krylov 子空间方法。
  • HPC：MPI/OpenMP/混合并行、网格划分与负载均衡、大规模算例下的带宽与缓存行为。
  • 性能与鲁棒性：在迭代轨迹、谱性质和工程可用性之间寻找平衡。

3. 软件工程与计算流程（Software Engineering & Workflow）

  • 代码架构：现代 C++ 风格的 CFD/CAE 求解器架构设计，模块化与可扩展性。
  • 测试与验证：单元测试、回归测试、算例库管理、物理量与残差的监控。
  • 计算流程：前处理（网格/边界条件）、求解控制、后处理与数据管理。
  • 工具化：脚本自动化、配置管理、结果复现与项目模板化。
  • 与商业软件/开源软件的协同：网格格式、后处理接口、数据交换。

4. 芯片与系统集成（Chips & System Integration）

  • 国产芯片架构理解：计算阵列、内存层次、带宽与延迟特性（在不涉密前提下）。
  • 算子映射：哪些算子适合上芯片（矩阵–向量、多右端、多网格），哪些应留在 CPU 侧。
  • 数据布局：稀疏矩阵存储格式、访存模式与 cache 友好布局。
  • 系统集成：从“CFD 求解器”到“软硬件一体计算平台”的接口设计与调度逻辑。
  • 性能对标：与常规 CPU/GPU 的加速比、能效、规模扩展的分析。

5. AI for CAE / Coding / Chips

  • AI + CAE：基于先验物理与数值分析的模型选择、参数调优、算例分类。
  • AI + Coding：借助 LLM 等工具加速 C++/Fortran 开发、重构、测试样例生成。
  • AI + 芯片：利用 AI 做瓶颈识别、参数搜索、任务调度策略探索（而非替代架构设计）。
  • 边界与约束：坚持物理约束、数值稳定性与工程可验证性作为第一原则。

PROJECTS · 项目与案例

以下只是示意结构。后续会逐步补充代表性的项目案例，每个案例都按照“问题背景 → 目标 → 技术方案 → 结果 → 反思”展开。

示例：航空发动机全流道仿真平台（Soft–Hardware Co-Design）

  • 问题背景： 多级风扇、压气机、燃烧室、涡轮的全流道气动与热力特性评估，对精度和计算时间都有刚性要求。
  • 目标： 在给定算力和时间预算下，比肩或超越商用软件的精度，并在国产芯片上获得可观的加速比。
  • 我的角色： 求解器设计与实现、隐式格式与稀疏矩阵求解优化、软硬件协同方案制定、测试与对标。
  • 技术要点： 网格与方程无量纲、RANS/LES 模型选择、Jacobian 构造、Krylov 求解器与预条件、MPI 并行、芯片侧矩阵算子部署。
  • 结果： 在典型算例上达到与商用软件相近的工程精度，加速比达到预期目标（细节后续在独立页面展开）。

示例：稀疏矩阵求解与芯片加速

  • 围绕 BiCGStab/CG 等迭代求解器，分析矩阵结构与谱性质，设计适合芯片的存储和算子调用模式。
  • 对比不同网格规模（例如 40 万 / 400 万 网格）下矩阵部分加速与整体 CFD 加速比的差异。
