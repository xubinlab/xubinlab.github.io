TECH STACK · 技术体系总览

这一页是我对「自己到底会什么」的一次拆解。每个板块后面都会有更细的专题页面和笔记链接， 方便别人快速判断：你在哪些层面和我聊，会比较对得上。

1. 物理与建模（Physics & Simulation Modeling）

从数学基础到控制方程，从湍流 / 多相 / 燃烧模型到具体工程场景（风扇、压气机、燃烧室、涡轮等）。 更多是「物理认识 + 建模取舍」层面的内容，而不是某个具体求解器的使用说明。

2. 数值方法、求解器与 HPC（Numerics, Solvers & HPC）

离散方法（FVM/FEM/DG）、时间推进、线性 / 非线性求解器、预条件、稳定性； 再到并行划分、负载均衡、缓存 / 带宽行为等——主要关注「算法如何落到机器上」。 （该部分页面后续逐步补全）

3. 软件工程与计算流程（Software Engineering & Workflow）

包括：软件调研与认识、自研代码的架构设计、编码规范与测试、算例库和 CI/CD， 以及一条完整的「前处理 → 求解 → 后处理 → HPC 作业」计算流程是怎样设计和演化出来的。

4. 芯片与系统集成（Chips & System Integration）

在不涉密的前提下，讨论如何从算子 / 数据布局的角度理解国产芯片架构， 以及把 CFD / CAE 求解器与自研芯片、整机系统打通的过程。 （该部分页面后续补全）

5. AI for Simulation / Coding / Chips

AI 在仿真中的作用（模型/工况推荐、结果判读）、在编码中的作用（代码生成、测试、重构）， 以及在芯片性能分析 / 调度策略探索中的可能性。更多是「怎么把 AI 用在现有工作流上」，而不是做大模型本身。 （该部分页面后续补全）
