PROJECTS

Aircraft Engine Full-Flowpath CFD Solver (2023-2024)

CFD Solver HPC Custom Chips

Background

Developing a comprehensive CFD simulation platform for multi-stage aircraft engine components (fans, compressors, combustors, turbines), requiring both high accuracy and computational efficiency for engineering design cycles.

My Role & Goals

Lead solver architect and developer, responsible for numerical scheme implementation, implicit solver optimization, and hardware-software co-design. Goal: Achieve engineering-acceptable accuracy within 2-4 hour time budgets on available computing resources.

Technical Highlights

  • Equation non-dimensionalization and RANS turbulence modeling for complex internal flows
  • Jacobian matrix construction and sparse linear system solving using Krylov methods
  • MPI parallelization with domain decomposition for multi-core scaling
  • Hardware-software co-design for custom chip deployment and memory optimization
  • Performance profiling and bottleneck analysis across different computing platforms

Results & Metrics

Achieved 3-5x acceleration on custom chips compared to traditional CPUs for representative test cases, while maintaining engineering accuracy within ±5% of commercial software results. Successfully deployed solver for production use in engine design workflows.

Reflection

The project highlighted the importance of algorithm-hardware co-design in modern CFD development. Key learning: performance optimization requires understanding both numerical methods and hardware architecture at multiple levels.

HPC Sparse Matrix Solver Optimization (2022-2023)

HPC Sparse Solvers Performance

Background

Implicit CFD solvers generate large sparse linear systems that dominate computation time. Traditional solvers struggle with irregular sparsity patterns and poor cache locality in engine flow simulations.

My Role & Goals

Solver algorithm researcher and optimizer, focusing on preconditioning strategies and parallel scaling. Goal: Improve convergence rate and parallel efficiency for industrial-scale engine simulations.

Technical Highlights

  • Analysis of sparsity patterns in CFD Jacobian matrices and their impact on solver performance
  • Implementation and comparison of algebraic multigrid (AMG) preconditioners
  • Development of problem-specific preconditioning strategies for turbomachinery flows
  • Memory bandwidth optimization and cache-aware data structures
  • Load balancing algorithms for heterogeneous computing environments

Results & Metrics

Reduced solver iteration count by 40-60% for compressor and turbine test cases through improved preconditioning. Achieved 85%+ parallel scaling efficiency on up to 1024 cores. Memory usage optimized by 30% through better data structures.

Reflection

This work demonstrated that understanding the physics of the problem (flow characteristics, boundary conditions) is crucial for developing effective numerical algorithms. Domain knowledge + numerical methods = better solvers.

AI-Assisted CFD Development Pipeline (2024)

AI Automation Software Engineering

Background

CFD development involves extensive parameter tuning, code refactoring, and performance analysis. Manual processes are time-consuming and error-prone, especially when deploying across different hardware platforms.

My Role & Goals

AI integration lead, exploring practical applications of AI tools in CFD workflows. Goal: Accelerate development cycles while maintaining code quality and numerical accuracy.

Technical Highlights

  • AI-assisted code generation and refactoring for numerical algorithms
  • Automated parameter optimization using machine learning surrogate models
  • Performance data analysis and bottleneck identification with AI tools
  • Code review and testing automation for CFD-specific patterns
  • Integration of AI tools into existing development workflows

Results & Metrics

Reduced code development time by 35% for new numerical features. Improved parameter tuning efficiency by 50% through AI-assisted optimization. Enhanced code quality with automated review tools catching 25% more potential issues.

Reflection

AI tools are most effective when used to augment engineering expertise rather than replace it. The key is knowing when and how to apply AI while maintaining physical understanding and numerical rigor.
